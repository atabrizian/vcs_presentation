What is a "Version Control System"?

It's a system that records changes to a file or set of files over time
so that you can recall specific versions later.

Why should you care?

  • Keep track of changes in files
  • Cooperate easier
  • Who did what, when did (s)he do it and why?
  • Maintain multiple versions of a project (branching)
  • Project Progress Control (Merge Policy)
  • Integration with other systems (hooks)

Centralized vs Distributed
In distributed version control rather than a single, central repository on which clients synchronize, each peer's working copy of the codebase is a complete repository.
